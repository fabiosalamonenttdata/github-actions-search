# Internationalization (i18n) Guide

This extension supports multiple languages with easy extensibility for adding new languages.

## Supported Languages

- **English** (en) - Default
- **日本語** (ja) - Japanese

## How Language Detection Works

The extension automatically detects the browser's language setting using `navigator.language`. If the browser language starts with "ja", Japanese is used; otherwise, English is the default.

The language preference is also stored in localStorage (`ghas-language`) for persistence.

## Architecture

The i18n system consists of three main files:

### 1. `src/content/util/i18n.ts`

Defines the translation keys and all translations:

```typescript
type TranslationKeys = {
  searchWorkflows: string
  searchPlaceholder: string
  closeButton: string
  noResults: string
  errorMessage: string
}

const translations: Record<Language, TranslationKeys> = {
  en: {
    /* English translations */
  },
  ja: {
    /* Japanese translations */
  },
}
```

### 2. `src/content/util/translation-provider.tsx`

React Context provider that manages language state:

```typescript
export const TranslationProvider: FC<{ children: ReactNode }>
export const useTranslation: () => TranslationContextType
```

### 3. `src/content/App.tsx`

Wraps the app with the TranslationProvider:

```typescript
<TranslationProvider>
  <LocalStorageProvider>
    <RepositoryProvider>
      <SearchDropdown repo={repo} />
    </RepositoryProvider>
  </LocalStorageProvider>
</TranslationProvider>
```

## Using Translations in Components

In any component, use the `useTranslation` hook:

```typescript
import { useTranslation } from "@/content/util/translation-provider"

export const MyComponent = () => {
  const { t } = useTranslation()

  return <button>{t("searchWorkflows")}</button>
}
```

## Adding a New Language

To add a new language (e.g., Spanish):

1. **Add to Language type** in `src/content/util/i18n.ts`:

```typescript
export type Language = "en" | "ja" | "es"
```

2. **Add translations** in `src/content/util/i18n.ts`:

```typescript
const translations: Record<Language, TranslationKeys> = {
  en: {
    /* ... */
  },
  ja: {
    /* ... */
  },
  es: {
    searchWorkflows: "Buscar Workflows",
    searchPlaceholder: "Ingrese palabras clave de búsqueda",
    closeButton: "Cerrar",
    noResults: "No se encontraron resultados.",
    errorMessage: "Error: Algo salió mal.",
  },
}
```

3. **Update language detection** in `detectLanguage()` if needed:

```typescript
export const detectLanguage = (): Language => {
  const browserLang = navigator.language || navigator.languages?.[0] || "en"
  if (browserLang.startsWith("ja")) {
    return "ja"
  }
  if (browserLang.startsWith("es")) {
    return "es"
  }
  return "en"
}
```

4. **Add tests** in `test/regression.spec.ts` for the new language if needed.

## Key Features

- Automatic browser language detection
- localStorage persistence
- Type-safe translation keys
- Easy to extend
- No external dependencies (pure React Context)
